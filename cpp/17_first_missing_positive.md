# 缺失的第一个正数（LeetCode #41）

## 题目描述

> 给定未排序整数数组，找出**未出现的最小正整数**。要求 O(n) 时间，O(1) 空间。

**示例：**
- 输入：`[3,4,-1,1]` → 输出：`2`
- 输入：`[7,8,9,11,12]` → 输出：`1`

---

## 梯度分析

### 🔵 第一层：布尔数组标记 O(n) 空间

将数组中出现过的正数标记到大小为 n+1 的布尔数组。空间 O(n)，不符合 O(1) 要求。

---

### 🟢 第二层：索引与值匹配 O(1) 空间

**核心洞察：** 若数组包含 `1..n` 中的数，就可以把 `x` 放到索引 `x-1` 处——利用索引本身做哈希表。

```
目标：将正数 x (1 <= x <= n) 放到位置 x-1
操作：遍历数组，若 nums[i] 在 [1,n] 且 nums[nums[i]-1] != nums[i]，交换两者
```

**图解 `[3, 4, -1, 1, 2]`（n=5）：**

```
初始: [3, 4, -1, 1, 2]
i=0: num=3, 在[1,5], nums[2]=-1≠3 → 交换 → [-1, 4, 3, 1, 2]
i=0: num=-1, 不处理
i=1: num=4, 在[1,5], nums[3]=1≠4 → 交换 → [-1, 1, 3, 4, 2]
i=1: num=1, 在[1,5], nums[0]=-1≠1 → 交换 → [1, -1, 3, 4, 2]
i=1: num=-1, 不处理
i=2: num=3, 在[1,5], nums[2]=3 ✅ 跳过（已在正确位置）
i=3: num=4, 在[1,5], nums[3]=4 ✅ 跳过
i=4: num=2, 在[1,5], nums[1]=-1≠2 → 交换 → [1, 2, 3, 4, -1]
i=4: num=-1, 不处理

最终: [1, 2, 3, 4, -1]
第一遍找 nums[i]!=i+1: i=4 → 答案=5
若全匹配 → 答案=n+1=6
```

**为什么只处理 `1 <= x <= n` 的数？**
- `x < 1` 或 `x > n` 的数无法找到正确位置（数组长度只有 n）
- 这保证了每个位置最多交换一次

**复杂度：**
- 时间：O(n)，每个元素最多交换一次
- 空间：O(1)

---

## 破题总结

> **利用数组索引做哈希表：将正数 x 放到索引 x-1 的位置。第一遍原地归位，第二遍找第一个不匹配的位置，即为缺失的最小正整数。**
