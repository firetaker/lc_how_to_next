# 滑动窗口最大值（LeetCode #239）

## 题目描述

> 给定数组 `nums` 和窗口大小 `k`，返回每个滑动窗口的最大值。

**示例：**
- 输入：`[1,3,-1,-3,5,3,6,7], k=3`
- 输出：`[3,3,5,5,6,7]`

---

## 梯度分析

### 🔵 第一层：暴力 O(n×k)

每个窗口遍历 k 个元素找最大值。总共 O(n×k)。

---

### 🟢 第二层：单调递减队列 O(n)

**核心洞察：** 滑动窗口的最大值，就是**队列中所有元素的最大值**。如果能维护一个从大到小单调递减的序列，每次取队首就是最大值。

**单调队列的运作原理：**

1. **右入**：新元素 `nums[i]` 从右侧入队前，弹出所有比它**小**的元素——它们永远不会再成为窗口最大值
2. **左出**：当队首元素离开窗口（索引 ≤ i-k）时，弹出队首

```python
dq = deque()  # 存索引，递减排列
for i in range(len(nums)):
    while dq and nums[dq[-1]] < nums[i]:
        dq.pop()      # 弹出较小元素
    dq.append(i)       # 入队
    if dq[0] <= i - k: # 队首已离开窗口
        dq.popleft()
    if i >= k - 1:
        result.append(nums[dq[0]])  # 队首即最大值
```

**图解 `nums=[1,3,-1,-3,5,3,6,7], k=3`：**

| i | nums[i] | 入队前 deque 值 | 入队后 deque 值 | 输出 |
|---|---------|----------------|----------------|------|
| 0 | 1 | [] | [0] | - |
| 1 | 3 | [0] | [1]（弹出0）| - |
| 2 | -1 | [1] | [1,2] | - |
| 3 | -3 | [1,2] | [1,2,3] | 3 |
| 4 | 5 | [1,2,3] | [4]（弹出1,2,3）| 5 |
| 5 | 3 | [4] | [4,5] | 5 |
| 6 | 6 | [4,5] | [6]（弹出4,5）| 6 |
| 7 | 7 | [6] | [7]（弹出6）| 7 |

**复杂度：**
- 时间：O(n)，每个元素最多入队出队各一次
- 空间：O(k)

---

## 破题总结

> **单调递减队列的队首就是当前窗口最大值——右入时弹出所有小于当前元素的旧索引（它们永远无法成为最大），左出时去掉离开窗口的索引，两个指针各走一遍即 O(n)。**
