# 轮转数组（LeetCode #189）

## 题目描述

> 将数组向右轮转 k 个位置（k 可能大于数组长度）。

**示例：**
- 输入：`[-1,-100,3,99], k=2` → 输出：`[3,99,-1,-100]`

---

## 梯度分析

### 🔵 第一层：创建新数组 O(n)

将尾部 k%n 个元素移到前面，补上剩余元素。空间 O(n)，不符合原地要求。

---

### 🟢 第二层：三次反转 O(1)

**核心洞察：** 轮转的本质是**把尾部 k%n 个元素移到前面**。通过三次原地反转实现：

```
原始: [1, 2, 3, 4, 5, 6, 7]  k=3
目标: [5, 6, 7, 1, 2, 3, 4]

Step 1: 整体反转
[7, 6, 5, 4, 3, 2, 1]

Step 2: 前 k 个反转 (k=3)
[5, 6, 7, 4, 3, 2, 1]

Step 3: 后 n-k 个反转 (n-k=4)
[5, 6, 7, 1, 2, 3, 4] ✅
```

```python
def rotate(nums, k):
    k %= n
    def reverse(l, r):
        while l < r:
            nums[l], nums[r] = nums[r], nums[l]
            l += 1; r -= 1
    reverse(0, n - 1)
    reverse(0, k - 1)
    reverse(k, n - 1)
```

**为什么三次反转是对的？**
- 反转的本质是**交换位置**：整体反转后前 k 个在正确后半段，后 n-k 个在正确前半段
- 再分别反转前后两部分，还原正确顺序

**复杂度：**
- 时间：O(n)
- 空间：O(1)

---

## 破题总结

> **三次反转实现原地轮转：整体反转 → 前 k 个反转 → 后 n-k 个反转。等价于把尾部 k 个元素移到前面，但无需额外空间。**
