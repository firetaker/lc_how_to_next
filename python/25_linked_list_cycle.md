# 环形链表（LeetCode #141）

## 题目描述

> 判断链表中是否存在环。

**示例：**
- `3→2→0→-4`，尾部连到节点 2 → `True`

---

## 梯度分析

### 🔵 第一层：哈希集合记录访问过的节点

遍历链表，若节点已在集合中则存在环。O(n) 时间，O(n) 空间。

---

### 🟢 第二层：Floyd 快慢指针 O(n) / O(1)

**核心洞察：** 快指针每次走两步，慢指针每次走一步。若存在环，两者必然在环内相遇。

```python
slow = fast = head
while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast:
        return True
return False
```

**为什么必定相遇？**
- 设环长为 c，快慢指针距离为 d
- 每一步快指针相对慢指针多走 1 步
- 经过 c-d 步后追上（相遇）

**图解 `3→2→0→4→2（回到2）`：**

```
初始: slow=3, fast=3

步1: slow=2, fast=0
步2: slow=4, fast=2（fast走了两步）
步3: slow=2, fast=4
步4: slow=0, fast=0（相遇）✅
```

**复杂度：**
- 时间：O(n)
- 空间：O(1)

---

## 破题总结

> **快慢指针同速赛跑：快指针两步、慢指针一步，若链表有环则快慢指针必然在环内相遇——匀速追赶模型。**
