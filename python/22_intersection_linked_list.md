# 相交链表（LeetCode #160）

## 题目描述

> 判断两链表是否相交，返回相交的第一个节点。

**示例：**
- 链表 A：`4→1→[8→4→5]`；链表 B：`5→6→1→[8→4→5]`
- 相交节点：`[8→4→5]`

---

## 梯度分析

### 🔵 第一层：哈希集合记录节点

遍历链表 A 存入 set，再遍历链表 B 找第一个在 set 中的节点。O(n+m) 时间，O(n) 空间。

---

### 🟢 第二层：双指针切换法 O(n+m) / O(1)

**核心洞察：** 指针 a 走完链表 A 后切换到链表 B，指针 b 走完链表 B 后切换到链表 A。若相交，两者走过的总路程相等，必然相遇。

```
a 的路径: A1→A2→A3 → B1→B2→B3
b 的路径: B1→B2→B3 → A1→A2→A3
```

数学证明：设 A 长度为 LA，B 长度为 LB，相交部分长度为 L。
- a 走过的距离 = LA + (LB - L)
- b 走过的距离 = LB + (LA - L)
- 两者相等 → 必在相交点或 nil 处相遇

```python
a, b = headA, headB
while a != b:
    a = a.next if a else headB
    b = b.next if b else headA
return a  # 或 b，相交点或 None
```

**图解：**

```
A: 4→1→8→4→5
         ↑________|
B: 5→6→1→8→4→5
         ↑________|

a 走: 4→1→8→4→5→5→6→1→(相遇8)
b 走: 5→6→1→8→4→5→4→1→(相遇8)
相遇节点: 8 ✅
```

**复杂度：**
- 时间：O(n+m)
- 空间：O(1)

---

## 破题总结

> **双指针走完各自链表后切换到另一条链表继续走——尾部等长消除后，两者走过的总距离相等，必然在相交点或 nil 处相遇。**
