# 最长连续序列（LeetCode #128）

## 题目描述

> 给定一个未排序的整数数组 `nums`，找出**最长连续序列**的长度（要求 O(n) 时间复杂度）。

**示例：**
- 输入：`[100, 4, 200, 1, 3, 2]`
- 输出：`4`（最长序列是 `[1, 2, 3, 4]`）

---

## 梯度分析

### 🔵 第一层：排序后遍历

最直接的想法是排序后遍历，统计连续序列的长度：

```python
nums.sort()
longest = cur_len = 1
for i in range(1, len(nums)):
    if nums[i] == nums[i-1] + 1:  # 连续
        cur_len += 1
    elif nums[i] == nums[i-1]:    # 重复，跳过
        continue
    else:                         # 断了，重置
        longest = max(longest, cur_len)
        cur_len = 1
```

**复杂度：**
- 时间：O(n log n)（排序主导）
- 空间：O(1)
- **问题：** 题目要求 O(n)，排序不符合要求

---

### 🟢 第二层：哈希集合（Set）+ 起点检测

**核心洞察：** 既然要求 O(n)，就不能排序。

将所有数放入 Set，利用 Set 的 O(1) 查询：

```python
num_set = set(nums)
longest = 0
for num in num_set:
    if num - 1 not in num_set:  # 只从序列起点开始扩展
        cur = num
        cur_len = 1
        while cur + 1 in num_set:
            cur += 1
            cur_len += 1
        longest = max(longest, cur_len)
```

**为什么只从「起点」开始？**

考虑序列 `[1,2,3,4]`：
- 如果从 1 开始：扩展到 4，统计长度 4 ✅
- 如果从 2 开始：扩展到 4，统计长度 3
- 如果从 3 开始：扩展到 4，统计长度 2

**不重复遍历的关键：** 只有当 `num - 1` 不在集合中时，`num` 才是序列的最小元素（起点）。从每个起点只扩展一次，每个元素最多被访问常数次。

**图解：**
```
nums = [100, 4, 200, 1, 3, 2]
num_set = {1, 2, 3, 4, 100, 200}

遍历 1: 1-1=0 不在 → 是起点 → 扩展: 1→2→3→4 → 长度=4
遍历 2: 2-1=1 在 → 不是起点 → 跳过
遍历 3: 3-1=2 在 → 不是起点 → 跳过
遍历 4: 4-1=3 在 → 不是起点 → 跳过
遍历 100: 100-1=99 不在 → 是起点 → 扩展: 100 → 长度=1
遍历 200: 200-1=199 不在 → 是起点 → 扩展: 200 → 长度=1

结果: max(4, 1, 1) = 4
```

**复杂度：**
- 时间：O(n)，每个元素最多被访问 2 次（放入 Set 1 次 + 扩展时最多访问 1 次）
- 空间：O(n)

---

## 破题总结

> **将数组全部放入 Set，从每个连续序列的最小元素（`num-1` 不在 Set 中）出发向后扩展，确保每个序列只遍历一次，达到 O(n) 时间。**
